<objective>
Remove the dead Vercel daily cron from `vercel.json` and tighten the `CLAUDE.md` Deployment prose to match (P4 — Low severity, but false-signal risk; SC-4).

The `crons` entry in `vercel.json` (`{ "path": "/api/cron/refresh", "schedule": "0 0 * * *" }`) 401s at the Vercel edge under Deployment Protection (Vercel Authentication) before the request ever reaches the route handler — and Vercel's cron dashboard reports that 401 as a *successful* invocation (false-green). During incident response this is actively misleading: it looks like a refresh path is working when it is not. QStash's 15-min POST is the only path that actually refreshes data under Deployment Protection.

Purpose: the Vercel cron dashboard stops showing a false-green for an invocation that never ran.
Output: `vercel.json` has no `crons` entry; the `CLAUDE.md` Deployment section states QStash is the sole working refresh path and explains the Vercel-edge-401 false-green.

This plan is pure config + docs — no runtime code. Fully independent of Plans 03-01, 03-02, 03-03. Wave 1.
</objective>
