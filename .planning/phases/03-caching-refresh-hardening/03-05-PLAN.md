<objective>
Implement the `/plan-eng-review` outcome for "TTL → 4h + ensure expiry + working reload": lower the cache hard-TTL from 24h to 4h, AND fix the two refresh paths that currently can't reload (Reddit, X) so the 4h expiry actually gets repopulated instead of 503ing.

Background (from the debugging session that preceded this plan):
- `reddit:hot`, `x:feed`, `hero:cross-platform` had not refreshed in 21h — confirmed via live Redis TTL inspection (the 5 healthy keys refreshed every 15 min; these 3 were frozen).
- Root cause, confirmed by Vercel runtime logs: `[reddit] Non-200 response ... 403 Blocked` (all 5 subreddits) and `[twitter] Non-200 ... 403 Forbidden` (all 7 accounts), both followed by `[cache] skipping empty write`.
- Reddit: `www.reddit.com/*.json` is datacenter-IP-blocked from Vercel (works from a residential IP — proven locally). X: the X fetcher CODE works (proven locally with the `.env.local` token, 200 / `rl-remaining=9999`); only Vercel's env token is stale/weaker.
- `hero:cross-platform` is a downstream casualty: `promoteHero()` only writes when there is a top-3-on-all-3-platforms candidate, and reddit/twitter contribute nothing to trending while their fetchers are dead. Once Reddit + X reload, hero recovers on its own. No direct hero code change needed.

Locked decisions (from the eng-review AskUserQuestion gates):
1. TTL role → short hard TTL, 503 on expiry (4h).
2. Issue-1 scope → bundle the fetcher fixes WITH the TTL change (one PR).
3. Reddit strategy → switch to the `.rss` (Atom) feed, parsed with `rss-parser`.
4. RedditPost model → remove `score` / `numComments` / `flair` (the type tells the truth); redesign the widget row.
5. X strategy → no code change; sync the working `X_BEARER_TOKEN` into Vercel env + redeploy.
6. Verification → build the full refactor, verify on deploy (the `.rss`-bypasses-Vercel-403 assumption is the accepted risk; see <failure_modes>).

Output: 4h hard TTL; Reddit fed by the Atom feed; `RedditPost` honest to what RSS provides; X reload fixed via env; docs updated.
</objective>

<data_flow>
Reddit refresh path — before vs after:

  BEFORE (broken from Vercel)
  cron → fetchRedditPosts → fetch www.reddit.com/r/{sub}/hot.json
                                   │
                                   └─ 403 Blocked (datacenter IP) ─→ [] ─→ cacheSet skips ─→ key ages → 503

  AFTER
  cron → fetchRedditPosts → fetch www.reddit.com/r/{sub}/hot.rss   (Atom)
                                   │
                                   ├─ 200 ─→ rss-parser ─→ normalizeRedditAtomEntry ─→ RedditPost[] ─→ cacheSet (ex 14400)
                                   └─ 403/non-200 ─→ [] ─→ cacheSet skips  ← ACCEPTED RISK (see failure_modes)

TTL clocks (unchanged mechanism, changed number):
  cacheSet  ──ex: 14400 (was 86400)──▶  Redis key
  cacheGet  ──age > CACHE_MAX_AGE──▶  stale flag (UNCHANGED: 15min/10min — soft freshness)
  cacheGet  ──key expired / null──▶  read route returns 503  (now reachable in 4h, was 24h)
</data_flow>
