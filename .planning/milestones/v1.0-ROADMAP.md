## Overview

v1.0 took AIP-Dash from a shipped data layer (SCRUM-36) + UI shell (SCRUM-37) to a full intelligence-layer dashboard: live cross-platform AI sentiment, true velocity-ranked trending via Redis ZSETs, an auto-promoted hero story when the same AI topic peaks on YouTube + Reddit + X simultaneously, a hardened cron refresh path that is observably honest under concurrency, and a 3-column scrollable feed UI that caps each platform at 15 items with a scroll-aware fade cue.

Phases 1 and 2 were retroactively imported into GSD after shipping (the work landed before GSD adoption). Phase 3 was added 2026-05-13 from a system-architect architecture review. Phase 4 was added 2026-05-14 from SCRUM-49.

### Phase 2: Reddit Free Fallback

**Goal**: Replace the Apify-based Reddit fetch with Reddit's free public JSON API. Eliminate the `Maximum cost per run is lower than actor start cost` failure mode that was dropping Reddit out of the cron DAG, restoring Reddit posts to the trending velocity calc (D2 ZSETs) and unblocking the cross-platform hero promotion threshold (D3 — needs Reddit in top-3 to fire).

**Depends on**: SCRUM-36 (data layer, shipped). Logically independent of Phase 1.

**Plans**: 1 (retroactive import via /gsd-import)

### Phase 3: Caching & Refresh Hardening

**Goal**: Make the cron refresh path observably honest and concurrency-safe. After this phase a cron run that writes nothing is visibly distinct from one that succeeded, two overlapping QStash deliveries cannot corrupt ZSET windows or double-spend the sentiment budget, and the Vercel cron dashboard no longer shows false-green for an invocation that never reached the route.

**Depends on**: Phase 1 (cron 3-tier DAG, sentiment engine, `cacheSet` helper, timeseries ZSETs) and Phase 2 (Reddit fetch path), both shipped. Operational hardening on shipped infrastructure — no new product surface.

### Issues Resolved

- ✅ Apify Reddit scraper failing in prod (`Maximum cost per run is lower than actor start cost`) → replaced with Reddit free JSON API (Phase 2); later switched to Reddit `.rss` in Phase 3 (commit `94c8a73`) because the JSON API is datacenter-IP-blocked from Vercel
- ✅ Cron summary lying about cache writes — `cacheSet` empty-array guard skip was reporting as `ok` → three-state outcome (Phase 3 SC-1)
- ✅ Concurrent QStash retries could race on ZSET writes + sentiment budget → distributed lock + atomic Lua check-and-consume (Phase 3 SC-2 + SC-3)
- ✅ Vercel daily cron showing false-green when 401-ing at the edge → removed (Phase 3 SC-4)
- ✅ Hardcoded `slice(0, 3-5)` in each feed widget capped UX at far below the available data → 15-item cap with scrollable body (Phase 4 SC-1)
- ✅ No render-time test infra for components → jsdom + @testing-library/react + @vitejs/plugin-react bootstrapped in Phase 4 Wave 0; 26 new component-render tests landed (102 → 128 total)
